# Milestone 8 Phase 8.2 Hotfix 1 - Runtime Profile Card Layout Polish

## Purpose

This hotfix improves the Runtime Profile Manager home layout introduced in Phase 8.2.

## Changes

- Increased the default application height from 800 to 900.
- Increased the minimum application height from 720 to 820.
- Added a styled scrollbar for runtime profile card overflow.
- Disabled horizontal scrolling in the profile card list.
- Kept startup action buttons in a fixed footer outside the scrolling profile-card area.
- Preserved the Phase 8.2 behavior where New Profile opens a neutral blank profile and Edit loads the selected profile.

## Scope

No runtime, provider, authentication, or service behavior changed.
